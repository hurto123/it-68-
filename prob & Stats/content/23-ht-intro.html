Hypothesis Testing

บทที่ 23: บทนำสู่การทดสอบสมมติฐาน

ทำความเข้าใจปรัชญาและขั้นตอนของการทดสอบสมมติฐาน ตั้งแต่งานกำหนดสมมติฐานศูนย์และสมมติฐานทางเลือก ไปจนถึงการตัดสินใจด้วยสถิติทดสอบ.

1. คำศัพท์หลัก

สมมติฐานศูนย์ (H₀)

คำกล่าวที่ถือเป็นค่าเริ่มต้น “ยังไม่มีการเปลี่ยนแปลง” มักใช้เครื่องหมาย = เช่น \(\mu = \mu_0\)

สมมติฐานทางเลือก (H₁)

สิ่งที่เราต้องการพิสูจน์ เช่น \(\mu > \mu_0\), \(p \ne p_0\)

ตรรกะการทดสอบ ถ้าข้อมูลขัดแย้งกับ H₀ มากพอ (สถิติทดสอบ “สุดโต่ง”) → ปฏิเสธ H₀ หากไม่ → ยังไม่ปฏิเสธ H₀

2. ขั้นตอนมาตรฐาน

1) กำหนด H₀ และ H₁ ระบุรูปแบบ one-tail / two-tail ให้ชัดเจน
2) เลือกระดับนัยสำคัญ (α) มักใช้ 0.05 หรือ 0.01 ขึ้นอยู่กับความเสี่ยงของ Type I error
3) คำนวณสถิติทดสอบ เลือกสูตรที่เหมาะกับข้อมูล (z, t, χ², F)
4) หาขอบเขตวิกฤติหรือ p-value เปรียบเทียบกับข้อมูล
5) ตัดสินใจและตีความ ระบุผลลัพธ์ในบริบทเชิงธุรกิจ/วิทยาศาสตร์

3. ความผิดพลาดประเภท I และ II

Type I Error (False Positive)

ปฏิเสธ H₀ ทั้งที่จริงๆ แล้ว H₀ ถูกต้อง ความน่าจะเป็นเท่ากับ α

Type II Error (False Negative)

ไม่ปฏิเสธ H₀ ทั้งที่จริง H₁ เป็นจริง ความน่าจะเป็นคือ β ซึ่งขึ้นกับขนาดตัวอย่างและ effect size

พลังการทดสอบ (Power) = 1 − β = โอกาสที่จะตรวจจับความแตกต่างเมื่อมีอยู่จริง

4. จำลองพื้นที่การตัดสินใจ

ลองปรับระดับนัยสำคัญ (α) และขนาดเอฟเฟกต์เพื่อดูการทับซ้อนของการแจกแจงภายใต้ H₀ และ H₁

α = 0.05
Effect size = 0.30σ

5. เปรียบเทียบการรายงานผล

องค์ประกอบ รูปแบบรายงานที่ดี
สถิติทดสอบ “t(28) = 2.31, p = 0.028”
ข้อสรุป “มีหลักฐานว่าค่าเฉลี่ยหลังการอบรมสูงขึ้น”
ขนาดเอฟเฟกต์ รายงานเช่น Cohen’s d หรือผลต่างค่าเฉลี่ยพร้อมช่วงความเชื่อมั่น

6. Checklist ก่อนตัดสินใจ

  • ✅ ระบุ H₀/H₁ ชัดเจน
  • ✅ เลือกระดับนัยสำคัญที่เหมาะสม
  • ✅ ตรวจสอบเงื่อนไขการใช้สถิติ (เช่น ความเป็นปกติ)
  • ✅ รายงานทั้ง p-value และขนาดเอฟเฟกต์
  • ✅ สื่อสารผลในบริบทของปัญหาจริง
