กลับหน้าหลัก

Guardrails ที่ควรรู้

  • ตรวจสอบสมมติฐาน Normality ก่อนใช้ Z/t test (ดูบท 14, 17, 25)
  • ขนาดตัวอย่างเล็กมาก (n<30) ควรเลือก t distribution เสมอเมื่อไม่รู้ σ
  • ค่า p ที่ใกล้เกณฑ์ควรสื่อสารพร้อม effect size และช่วงความเชื่อมั่น
  • สำหรับ Chi-square ควรแน่ใจว่า expected count ≥ 5 อย่างน้อย 80% ของเซลล์
  • Power analysis ควรใช้ variance สมมติฐานที่ conservative เพื่อลดความเสี่ยง

Correlation & Regression

  • ค่า r สูงไม่แปลว่ามีสาเหตุ → ตรวจ confounder และ reverse causality (ดูบท 32, 34)
  • ก่อนฟิต SLR/MLR ให้ตรวจ scatter plot และ multicollinearity ด้วย VIF หรือ heatmap
  • รายงานทั้งค่า β, SE, และช่วงความเชื่อมั่น พร้อม residual diagnostics (บท 33–34)
  • ใช้ Regression Playground เพื่อตรวจสมมติฐานแบบโต้ตอบ

ANOVA & Chi-square

  • One-way ANOVA ต้องการกลุ่มที่สุ่มอิสระและความแปรปรวนใกล้กัน (บท 35)
  • ถ้า SSW สูงผิดปกติ → ตรวจ outlier และพิจารณาแปลงข้อมูลก่อนสรุปผล
  • Chi-square Independence ควรเช็ก standardized residual เพื่อหาหมวดที่แตกต่าง (บท 36)
  • ใช้ ANOVA Calculator และ Chi-square Tool เพื่อทวนขั้นตอน

Resampling & Bayesian

  • Bootstrap ใช้ได้ดีเมื่อ analytic SE ยุ่งยาก แต่ข้อมูลควรสะท้อนประชากรจริง (บท 37)
  • แยก seed และจำนวนรอบ B ไว้อย่างโปร่งใส พร้อมบันทึกเครื่องมือที่ใช้ (ดู Bootstrap Lab)
  • Bayesian analysis ต้องระบุ prior อย่างชัดเจน และทดสอบความไวของผลต่อ prior (บท 38)
  • DoE ควรวาง randomization/blocking ไว้ใน protocol และตรวจสอบการเก็บข้อมูล (บท 39)
